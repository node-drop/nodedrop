# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: security@nodedrop.io (or create a private security advisory on GitHub)

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Initial response**: Within 48 hours
- **Status update**: Within 7 days
- **Fix timeline**: Depends on severity

## Security Best Practices for Users

1. **Keep Node-Drop updated** to the latest version
2. **Use strong passwords** for admin accounts
3. **Secure your `.env` file** - never commit secrets
4. **Use HTTPS** in production
5. **Restrict database access** to trusted networks
6. **Review workflow permissions** regularly
